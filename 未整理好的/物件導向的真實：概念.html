物件導向的真實：概念

理念

物件導向編成簡稱：「OOP」，全部寫出來是「Object Oriented Programming」。 有時候會說「OO」就是物件導向的意思。
The world is created by objects. Therefore, if the program should successfully simulate events in this world, it has to know how to work with objects. Rudolf Pecinovsky: OOP - Learn Object Oriented Thinking & Programming P35
物件導向就是模仿真實世界物件的行為。 世界是由物體創造的。 因此，如果程式要能夠成功模擬 這個世界上的事件，它必須知道該物件有如何的行為。 好像養貓必須知道貓的習慣與一切行為，牠會叫、會吃、會睡、會餓。 Python、C#、Java、JavaScript、Rust很現代的語言都是物件導向的設計。

核心邏輯

其中必須要明白的是，物件導向編成會使用類別來描述物件，英語叫做「class」。 相對於類別的是實例，英語叫做「instance」。 他們的區別是類別是我們怎麼稱呼實例，歸類實例的方式。 很像生物學，橘貓、黑貓、白貓都是貓科動物，貓科動物是類別。 實例比較像是我們家的貓，叫做「小黑」，是一隻黑貓。 以下是描述類別時的程式碼： class Cat {} 以下是描述實例時的程式碼： Cat littleBlack = new Cat(); 所以，當我們在閱讀OOP特質的程式碼時， 只要找到類別，就可以看見這個物件的行為。 當你遇到需要解決的問題，就要找到對應的物件，執行物件的行為以解決我們的問題。
