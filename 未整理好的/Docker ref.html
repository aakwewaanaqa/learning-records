Docker 是什麼？

Docker 是一個平台，其包含 Hub、客戶端指令 CLI、桌面應用程式 EXE，是用來減少開發到部屬時的工作時間。
需要學習它的 CLI 指令、Dockerfile 語法、compose.yaml 語法，以完成部屬作業。
但其中有許多的範例可以參考、技術文件也完整，所以學習起來不難。

核心概念

  1. Container

    稱作「容器」，是一個獨立的作業程序，是 Image 建置後的產物。
    它是獨立的、隔絕的、自我乘載的、可移動的。

    關於虛擬機與容器的比較：
    虛擬機是另一個完整的作業環境，容器只是一個獨立的作業程序。

  2. Image

    稱作「映像檔」，就是可以成為容器的模板。
    映像檔是許多已經被標準化準備啟用的模板，是由許多提供功能的社群、官方維護。
    它是唯獨不可修改、由許多層級組成，層級寫在 Dockerfile 中。

  3. Docker CLI

    是 Docker 在我們電腦上能夠執行指令的程式。

    # 常用的是：
    docker run # 用來執行 Dockerfile
    docker compose up --build # 用來執行 compose.yaml

  4. Host

    就是本機，執行 Docker CLI 的環境、用網路抓取 Image 的環境。

    所以建置過程需要有網路😂
    不要忘記要連上網路。

  5. compose.yaml

    用來設定多個 Container 之間的關係。

  6. Dockerfile

    Image 中指示要增加層級的檔案。

架構關係

compose.yaml ➡️ Dockerfile
Dockerfile ➡️ Docker CLI
Docker CLI ➡️ 本機網路 ➡️ Docker Hub ➡️ Image
Image ➡️ Container
